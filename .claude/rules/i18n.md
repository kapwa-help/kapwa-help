# Internationalization

Three languages: English (`en`), Filipino (`fil`), Ilocano (`ilo`).

## How It Works

- **Detection**: Language from URL path (`/:locale/...`), configured in `src/i18n.ts`
- **Fallback**: Missing keys fall back to English
- **Interpolation**: `{{variable}}` syntax
- **In components**: `const { t } = useTranslation(); t("Dashboard.hero")`
- **Key structure**: Two-level nested `Namespace.key`. Namespaces: `App`, `Navigation`, `Dashboard`, `Stories`, `SubmitForm`

## Adding Keys

1. Add to `public/locales/en/translation.json` (English is source of truth)
2. Run `npm run translate` — incremental, only translates new/untranslated keys
3. Upload to Crowdin for human review

## Translation Script (`scripts/translate.ts`)

- Uses `google-translate-api-x` (free, no API key)
- Preserves `{{interpolation}}` variables during translation
- Skips keys already translated, brand names, pure interpolation, URLs
- Language mapping: `fil/` -> Google Translate `tl` (Tagalog), `ilo/` -> `ilo`

## Locale Routing

`RootLayout` syncs i18n language with URL param. Language switcher in Header navigates to `/:locale` routes.
