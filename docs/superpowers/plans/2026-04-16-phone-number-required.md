# Phone Number Required (Need) + Optional Phone (Hazard) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the Phone Number field required on the Need submission form. Add an optional Phone Number field to the Hazard submission form, persist it to the database, and display it as a tap-to-call `tel:` link in the hazard detail panel.

**Architecture:** Front-end form changes plus a single additive DB column on `hazards`. No RPC changes. The needs path already supports `contact_phone` end-to-end — this plan only flips the UI from optional to required for needs. The hazards path gets a new field threaded through the insert interface, query selector, mapper, form, and detail panel. Existing offline-outbox flow is unchanged because the new column is nullable.

**Tech Stack:** Vite + React 19 + TypeScript, react-i18next, Supabase (Postgres), Tailwind v4, Vitest + Playwright.

**Spec:** `docs/superpowers/specs/2026-04-16-phone-number-required-design.md`
