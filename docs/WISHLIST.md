# Product Wishlist & Future Features

## Accessibility & Inclusivity
- [ ] **Hover-to-Speech (TTS) Integration**
    - **Description**: Use the native Web Speech API to read aloud UI elements (cards, buttons, summaries) when a user hovers over them.
    - **Benefit**: Zero-bandwidth, zero-latency accessibility for visually impaired or high-stress users.
    - **Implementation**: Custom `useTTSHover` hook + Header toggle.
- [ ] **Intelligent Map Annotations**
    - **Description**: Conversational audio summaries of map pins (e.g., "Verified need in San Juan: Water and Blankets").
- [ ] **Dynamic Language Support**
    - [x] English / Tagalog / Ilocano framework (i18next).

## UI/UX Refinements
- [ ] **Theme Persistence**: Save user's preferred theme (Coastal Escape, Deep Sea, etc.) in local storage.
- [ ] **Offline Mode Indicators**: Clearer visual cues when data is being served from local cache during connectivity drops.

## Data & Coordination
- [ ] **Real-time SMS Integration**: Feeding SMS reports directly into the "Needs" pipeline.
- [ ] **Volunteer Logistics Dashboard**: Heatmaps of volunteer density vs. active needs.
